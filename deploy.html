Deploy & configuration

Everything the one-command installer sets up for you — and the knobs you can turn.

Prerequisites

  • A domain with a DNS A record pointing at your server's public IP.
  • A Debian/Ubuntu server with root access (the installer uses apt).
  • Inbound TCP 80 and 443 open — port 80 is needed for the automatic Let's Encrypt certificate.
  • Recommended: Cloudflare in front to hide your origin IP.

One-command install

Run this on the server as root. It installs dependencies, asks for your domain, requests a Let's Encrypt certificate (self-signed fallback), renders the nginx config, generates the access token, writes the systemd units, and starts everything.

wget -qO- https://raw.githubusercontent.com/endlessdetour/Callsign/main/deploy/install-server.sh | sudo bash

Installer options

Environment variable Effect
CALLSIGN_DOMAINYour domain (asked interactively if omitted).
CALLSIGN_BRANCHInstall from a specific branch (defaults to main).
CALLSIGN_TRUST_CLOUDFLARESet 1 to keep the Cloudflare-only origin gate in nginx.
CALLSIGN_REQUEST_SSL_CERTSet 0 to skip Let's Encrypt and force a self-signed cert.
CALLSIGN_LE_EMAILOptional email for the Let's Encrypt registration.

Server environment file

The installer writes /etc/proxy-server.env (mode 0600). Key settings:

# Mandatory access token for all control/tunnel HTTP requests.
CALLSIGN_ACCESS_TOKEN=<generated-strong-random-token>

# Control endpoint used by the tunnel service for token validation.
CONTROL_VALIDATE_URL=http://127.0.0.1:5000/api/v1/validate

# Public WSS endpoint returned to clients during bootstrap.
CALLSIGN_TUNNEL_PUBLIC_URL=wss://<your-domain>/connect-ws

# Keep this path consistent across services and the reverse proxy.
CALLSIGN_TUNNEL_PATH=/connect-ws

# Tunnel mode: tun = real forwarding, echo = debug-only.
CALLSIGN_TUN_MODE=tun

# Session TTL for the control service (seconds).
CALLSIGN_SESSION_TTL=3600

nginx reverse proxy

The installer renders this from the example template, substituting your domain and certificate paths. Only 443 is open; everything else returns 444.

upstream control_plane { server 127.0.0.1:5000; }
upstream tunnel_plane  { server 127.0.0.1:8443; }

server {
    listen 443 ssl http2;
    server_name <your-domain>;

    ssl_certificate     /etc/letsencrypt/live/<your-domain>/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/<your-domain>/privkey.pem;

    # Optional Cloudflare-only origin gate:
    if ($from_cloudflare = 0) { return 444; }

    location /api/ {
        proxy_pass http://control_plane;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
    }

    location = /login {
        proxy_pass http://control_plane/login;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
    }

    location /connect-ws {
        proxy_pass http://tunnel_plane/connect-ws;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 3600;
    }

    # Everything else is closed.
    location / { return 444; }
}

systemd services

Three units are installed and enabled, so the stack survives reboots:

Unit Role
proxy-control.serviceControl plane (Flask under gunicorn) + admin console.
proxy-tunnel.serviceWebSocket tunnel / data plane.
callsign-nat.serviceSets up NAT / forwarding for the private-network tun interface.
 
# Check status / logs
systemctl status proxy-control.service proxy-tunnel.service callsign-nat.service
journalctl -u proxy-control.service -f

First login

Open https://<your-domain>/login. The installer prints the seeded admin credentials and writes them to /etc/callsign/initial_admin_credentials.txt (root only, mode 0600). On first login you must set a new admin username and password; the seeded account is then disabled.

Connect a client

Download the Windows client from Releases, enter your Control URL (https://<your-domain>) and the access token, then connect. The token is encrypted at rest with DPAPI.
