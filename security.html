Security

Strong, sensible defaults that you don't have to configure. Here's exactly what's protecting your deployment.

Authentication

  • Passwords are hashed with pbkdf2_sha256 (120,000 rounds) plus a per-user salt.
  • Credential comparison is constant-time, and login timing is equalized to prevent username enumeration.
  • Forced first-login credential change: the seeded admin account is disabled once you set your own admin.

Sessions & tokens

  • Session tokens are server-stored and generated with secrets.token_urlsafe.
  • Each device gets its own private IP lease, so addresses never collide.
  • Resetting a user's password revokes their sessions and forces a change on next login.

Network & edge

  • The control plane runs under gunicorn behind nginx; only HTTPS on 443 is exposed and everything else returns 444 (closed).
  • Tunnel source-IP anti-spoofing validates the real peer address.
  • Optional Cloudflare-only origin gate (nginx geo on the real peer address) so only Cloudflare can reach your origin.

Hardening headers

Every response carries security headers and the server fingerprint is stripped:

  • Content-Security-Policy
  • Strict-Transport-Security (HSTS)
  • X-Frame-Options, X-Content-Type-Options, Referrer-Policy
  • server_tokens off and the default nginx welcome page is closed.

Client at rest

  • The Windows client encrypts its access token at rest with DPAPI, so the saved profile file does not contain a plaintext token.
This is a hardened prototype. For production, move from a single shared access token toward mTLS and signed short-lived tokens — see the Architecture page.
